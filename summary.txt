# AuraGuard — Project Summary

## Stack
- **Frontend**: Next.js 16, React 19, Tailwind CSS 4, TanStack Query 5, Lucide icons
- **Backend**: Supabase (Postgres, Auth, Storage), Next.js API routes
- **Auth**: Hardcoded demo creds pre-filled for hackathon
- **Data**: All fetching via TanStack Query — shared cache keys (`['folders']`, `['files']`, `['scan-results']`, etc.)
- **Scanner engine**: Native TypeScript in `web/lib/engine/` — cheerio, exifreader, @google/generative-ai
- **Forge (watermarking)**: Python in `engine/forge/` — PIL, piexif, mutagen
- **Test-shop**: FastAPI server in `test-shop/`, deployed on Railway

## Pages

- `/dashboard/assets` — Folder grid, upload modal
- `/dashboard/assets/[slug]` — Per-folder file grid, file detail modal
- `/dashboard/alerts` — Blog-style scan results list
- `/dashboard/alerts/[id]` — Alert detail with impact summary
- `/dashboard/scan-logs` — Raw scan results table
- `/dashboard/system` — Live service health checks

## Key Decisions
- Dark theme: canvas #000, surface #111, elevated #1A1A1A, borders #333/#444
- Folder cards look like folders (tab + body)
- Files display as square thumbnails in wrapping grid
- File extensions stripped in UI via `stripExt()` helper
- Image/video viewer: flex-col modal, `max-h-[85vh]`, fixed bars
- Scanner feed: single rolling line, fade in/out, tick/warning indicator
- Post-scan: green "Nothing found" or amber "X matches found"
- Loading: simple ternary — no crossfade
- Back links inline below titles — no separate header bar
- Alerts: blog-style text layout, flowing summary, minimal contact chips
- AI summaries: neutral reporter tone via Gemini 1.5 Flash
- Never call `el.remove()` — let React handle DOM

## Scanner Engine (TypeScript — `web/lib/engine/`)
- `config.ts` — env vars (SUPABASE_URL, SERVICE_KEY, GEMINI_API_KEY, domains)
- `crawler.ts` — cheerio-based HTML scraper, resolves relative URLs, extracts emails
- `scanner.ts` — exifreader EXIF parsing, detects `AURAGUARD_TRACKING_ID=AG-xxxx`
- `ai.ts` — Google Gemini client for impact summaries
- `db.ts` — Supabase lookups + scan_results inserts (service key)
- `detector.ts` — orchestrator: crawls PRE_CONFIGURED_DOMAINS, scans images, looks up tracking IDs in DB, records matches
- `web/app/api/scan/route.ts` — POST handler, calls `detectAll()` directly

## Scanner Flow
1. Crawl each domain in PRE_CONFIGURED_DOMAINS (currently `http://localhost:5000`)
2. For each `<img>` on the page, download and scan EXIF for `AURAGUARD_TRACKING_ID=AG-xxx`
3. If found, look up that tracking_id in the `files` Supabase table
4. If matching file exists, generate AI impact summary
5. Insert `scan_results` record (deduped by tracking_id + source_url)

## Forge Pipeline (Python — `engine/forge/`)
- `worker.py` — polling loop: queries files WHERE status = "processing"
- Downloads original from Supabase Storage
- Checks for existing tracking ID (re-upload detection)
- Stamps EXIF/metadata with new tracking ID
- Re-uploads stamped file, marks as "secured"
- Heartbeat upsert every poll cycle

## Supabase Schema
- `folders`: id, name, date, description, brand, campaign, category, price, status, thumb, user_id
- `files`: id, folder_id, name, type, size, thumb, url, storage_path, status, tracking_id, error_log, user_id
- `scan_results`: id, tracking_id, file_id, user_id, matched_url, matched_image_url, page_title, site_email, impact_summary, detected_at
- `service_heartbeat`: service_name, last_seen
- RLS: folders/files by auth.uid(); scan_results select/insert for own user_id

## Previous Fixes
- Back button UX: removed duplicate top bar, inline `← Alerts` below title
- AI impact summaries: neutral reporter prompt (not brand protection analyst)
- Scanner: always crawls domains, looks up tracking IDs from DB (no longer requires pre-"secured" files)
- PATH resolution: `findUp()` helper in API route
- Scanner deps: Pillow, piexif, mutagen in test-shop venv + requirements.txt
- Video thumbnails: frame capture at 1s via canvas, fallback play icon if CORS blocks

## Important Notes
- Port 3000 is in use — do not kill it
- Scanner needs Forge worker to have stamped files for tracking IDs to exist in DB
- Forge needs SUPABASE_SERVICE_KEY + GEMINI_API_KEY in `engine/.env`
- Test-shop uses `shop` schema (not `public`) — `shop.items` table + `shop-media` bucket
- Test-shop media served via FastAPI proxy for correct Content-Type
- EXIF watermark in `UserComment` with prefix `AURAGUARD_TRACKING_ID=`
